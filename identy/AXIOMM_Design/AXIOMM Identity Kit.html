How to use

GitHub README
SVGs render natively in GitHub markdown. Place the lockup at the top of your README.md.
![AXIOMM](axiomm_wave_lockup.svg)
PyPI · pyproject.toml
PyPI doesn't render SVG — use the 128 px PNG. Add to your pyproject.toml under [project].
# pyproject.toml [project] name = "axiomm" [tool.setuptools.package-data] axiomm = ["axiomm_wave_128.png"]
MkDocs (docs/)
Place the icon SVG in docs/ and reference it in mkdocs.yml.
# mkdocs.yml theme: name: material logo: axiomm_wave_icon.svg favicon: axiomm_wave_icon.svg
Sphinx (conf.py)
For Sphinx docs using the Furo or PyData theme.
# conf.py html_logo = "_static/axiomm_wave_icon.svg" html_favicon = "_static/axiomm_wave_32.png"
HTML embed
Inline SVG for pixel-perfect rendering at any size in web docs or notebooks.
<img src="axiomm_wave_icon.svg" alt="AXIOMM" width="48" height="48">
Jupyter / notebook
Display the mark in an output cell or as a header in your notebook.
from IPython.display import Image Image("axiomm_wave_icon.svg", width=64)

Usage rules

✓ Do
Use the SVG for all web contexts — it scales to any size
Use the dark lockup on backgrounds darker than #7A4858
Keep clear space equal to the mark height on all sides
Use axiomm_wave_128.png for PyPI and conda-forge
✗ Don't
Don't stretch or distort the mark asymmetrically
Don't change the gradient colours or direction
Don't place the light lockup on coloured backgrounds
Don't use the mark below 16 px without fallback to flat #C8326A
